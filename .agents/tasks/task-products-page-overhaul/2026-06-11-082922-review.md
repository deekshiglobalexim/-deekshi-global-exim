# Products page overhaul: premium card redesign with Framer Motion

The products page was rewritten from a static server component into a client-side interactive page with Framer Motion staggered entrance animations, redesigned product cards with export trust badges, corrected image URLs, reordered categories (Rice first), and a responsive 1/2/3-column grid. A companion `layout.tsx` was introduced to preserve metadata exports that can't coexist with `'use client'` in the same file.

Watch for: The `scrollbar-hide` utility class used on the sticky navigation bar is not defined anywhere in the project's CSS or Tailwind config (confirmed). Basmati Rice carries an "AP & Telangana Sourcing" badge despite being primarily a Punjab/Haryana product (likely factual inaccuracy). The Non-Basmati Rice image (`photo-1594054901127`) may not clearly depict non-basmati/parboiled rice varieties (possible — needs visual verification).

## High-level view

The server-to-client component split is handled correctly: metadata moves to `layout.tsx` and the page gains Framer Motion orchestration via `containerVariants`, `cardVariants`, and `headerVariants` with `whileInView` scroll-based triggers. Variants are defined outside the component to avoid re-creation on renders.

The product data was curated down from 24 items (including generic "Seasonal" catch-all entries) to 18 specific products across four categories ordered by business priority: Rice, Fruits, Vegetables, Spices. Each card carries three image-overlay badges (Export Quality, Bulk Supply, Global Export), a conditional "AP & Telangana Sourcing" pill for rice, and a trust footer — but the badge density creates visual crowding on mobile viewports.

The responsive grid uses `grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8` with `flex flex-col h-full` on cards and `flex-grow` on the description paragraph for equal-height cards. Image containers use `aspect-[4/3]` with Next.js `fill` layout. The pattern is solid; the only implementation gap is a `scrollbar-hide` class referenced on the sticky category nav without any backing CSS definition.

## Non-Basmati Rice image and deduplication

The original page used the same Unsplash photo (`photo-1536304993881`) for both Basmati and Non-Basmati rice. This change assigns Non-Basmati a distinct image (`photo-1594054901127`), which eliminates the duplication. However, this new photo ID should be visually verified — it's not from the well-known set of rice product photography on Unsplash and may depict a food preparation scene rather than raw export-grade rice.

## Basmati sourcing badge: geographic inaccuracy

The `badges` array on all three rice products includes `'AP & Telangana Sourcing'`. The card renders this conditionally as a `<MapPin>` pill. While Sona Masoori rice is indeed an Andhra Pradesh/Telangana variety, Basmati rice is primarily cultivated in the Indo-Gangetic plains (Punjab, Haryana, Himachal Pradesh, Uttarakhand). Labeling Basmati with "AP & Telangana Sourcing" is factually misleading to international buyers who know Basmati geography. The same concern applies to Non-Basmati (IR-64), which is grown across multiple states — the badge implies exclusive AP/Telangana sourcing that doesn't reflect reality for all varieties listed.

## Trust badge density on image overlays

Each product card image carries three overlapping badge elements:

```
[top-left]    Global Export        [top-right]   Export Quality
[bottom-left] Bulk Supply Available
```

On a 4:3 image at mobile width (~350px), the top-left and top-right badges are roughly 10px from their respective edges, leaving roughly 80px of clear image between them. The overall effect is that approximately 40% of the visible image area is occupied by overlay elements at small viewports. A pragmatic fix: hide the "Global Export" badge on small screens with a `hidden md:flex` class, which preserves all the trust signaling on desktop where there's room for it.

## Missing scrollbar-hide utility

The sticky category navigation uses `scrollbar-hide` in its class list:

```tsx
<div className="flex gap-2 overflow-x-auto py-4 scrollbar-hide">
```

No `.scrollbar-hide` class exists in `globals.css`, `tailwind.config.ts` doesn't define it, and no `tailwind-scrollbar-hide` plugin is installed (`package.json` has no scrollbar-related dependency). The class is silently ignored. On Android Chrome the horizontal overflow area will show a native scrollbar; iOS hides it by default. Fix with a utility rule:

```css
.scrollbar-hide {
  -ms-overflow-style: none;
  scrollbar-width: none;
}
.scrollbar-hide::-webkit-scrollbar {
  display: none;
}
```
