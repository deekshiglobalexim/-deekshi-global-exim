# Products page overhaul: images, content, ordering, and card UI

The products page was rewritten to match a priority product list, replacing all images with Unsplash URLs, removing Papaya and Okra, reordering categories (Rice first), rewriting descriptions for premium export positioning, and refining the card UI with larger padding, elevated shadows, wider gaps, and full-length descriptions. The sourcing badges now show specific Indian regions for every product. Watch for: the Non-Basmati Rice image URL deviates from the spec-suggested URLs (confirmed), the product count totals 16 rather than the acceptance criteria's stated 15 (confirmed), and the shadow class swap removes the brand-tinted premium shadows from the parallel palette task (confirmed).

## High-level view

The category and product ordering follows the required priority: Rice, Fruits, Vegetables, Spices. Papaya and Okra are gone. The remaining product list totals 16 items (3+4+4+5), which contradicts the acceptance criterion headline of "exactly 15 products" — though the criterion's own breakdown (3 rice, 4 fruits, 4 vegetables, 5 spices) sums to 16, so this appears to be a spec typo rather than an implementation error.

Non-Basmati Rice uses `photo-1550358864-518f202c02ba` (loose white rice grains) instead of the two spec-suggested URLs. The choice avoids reusing the Sona Masoori photo and is visually appropriate, but deviates from the written spec.

The shadow class swap from `shadow-premium` / `shadow-premium-lg` to Tailwind's generic `shadow-xl` / `shadow-2xl` removes the gold-tinted rgba shadows defined in `tailwind.config.ts` during the parallel premium palette task. This is a subtle visual regression that loses brand-colored shadow edges.

## Non-Basmati Rice image choice

The feature spec suggests either reusing the Sona Masoori photo (`photo-1586201375761-83865001e31c`) with a different crop parameter, or using `premium_photo-1723869067492-f2b3c38e15c1` for a parboiled-rice look. The implementation chose a third option — `photo-1550358864-518f202c02ba` — which shows a pile of white rice grains. This avoids same-photo reuse (a good instinct) but deviates from the spec. The `plus.unsplash.com` domain for the premium photo is already whitelisted in `next.config.ts`, so using it would not require config changes.

## Shadow class swap conflicts with palette task

The diff changes `shadow-premium` to `shadow-xl` and `hover:shadow-premium-lg` to `hover:shadow-2xl`. The custom premium shadows in `tailwind.config.ts` use gold-tinted rgba values (`rgba(198, 138, 46, ...)`) to give cards a warm brand-colored shadow edge — this was specifically implemented in the `task-global-premium-palette` feature. Standard Tailwind `shadow-xl` / `shadow-2xl` produce neutral gray shadows. If the intent was to simplify, the custom shadow definitions in tailwind config become dead code. If the intent was to keep the premium feel, the original custom classes should be restored.
