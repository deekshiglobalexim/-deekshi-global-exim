# Export-business messaging overhaul and new homepage sections

The change pivots the website's voice from a generic agricultural-products site to an export-focused business targeting international buyers. It upgrades the Hero, Stats, WhyChooseUs, Products page, and Footer with export-oriented copy and badges, and adds two entirely new homepage sections (ExportProcess, InternationalMarkets). All new UI follows the existing design system (navy + gold palette, Framer Motion stagger animations, Lucide icons, `card-premium` / `shadow-premium-lg` CSS classes).

Watch for: conflicting Tailwind `text-lg` and `text-sm` classes on all three footer column headings (confirmed); every product card getting identical "Export Grade" and "Bulk Supply" badges regardless of actual product qualification (confirmed); the "100% Quality Commitment" stat counting up from 0 to 100 with the existing CountUp animation, which may feel odd for a percentage that isn't a real metric (likely).

## High-level view

The Hero section now leads with "Trusted Global Export Partner" at very large type sizes and two CTAs that both link to `/contact`. The two-button treatment implies distinct actions but delivers the same destination, which dilutes the visual differentiation.

Stats were changed from demonstrable numbers ("5+ Active Certifications") to a mix of real metrics and aspirational claims ("100% Quality Commitment"). The CountUp animation still drives all four values, so the user sees 0 animate to 100 before appending `%` — fine for numeric stats but reads like a loading indicator for what is essentially a slogan.

The Products page adds "Export Grade" and "Bulk Supply" pill badges to every single product card image, and gives the Rice category a gold top-border with a "Priority Export Category" badge. The badges are unconditional — every product from Sona Masoori to Onion Powder receives identical labels regardless of whether those claims are accurate per-product. A badge that appears on everything communicates nothing.

The Footer gains a tagline paragraph and a "Request Quotation" CTA button, but the restyled column headings carry both `text-lg` and `text-sm` in the same class string, creating a Tailwind conflict where the rendered size depends on stylesheet generation order.

## Footer heading Tailwind conflict

The diff adds `font-bold`, `tracking-wide`, `uppercase`, and `text-sm` to footer `<h4>` elements that already carried `text-lg` from the previous code. The resulting class string is:

```
font-display font-bold text-gold-500 mb-5 text-lg tracking-wide uppercase text-sm
```

In Tailwind, when two utilities target `font-size`, the one whose generated CSS appears later wins. In practice this renders as `text-sm` on all three headings (Quick Links, Our Products, Contact Us), which is likely unintentional since the original design used `text-lg`. Fix: remove `text-sm` if you want large headings, or remove `text-lg` if small uppercase headings are the intent.

## Unconditional product badges

Every product card receives:

```tsx
<span className="...">Export Grade</span>
<span className="...">Bulk Supply</span>
```

These are hardcoded inside the `.map()` with no conditional logic. The badges communicate nothing that differentiates products from each other — a badge that appears on everything is visual noise, not information. If the intent is eventual per-product differentiation, adding `exportGrade?: boolean` and `bulkSupply?: boolean` to the product data model now would prevent a future refactor.

## Duplicate CTA destination in Hero

Both "Request Quotation" and "Contact Export Team" link to `/contact`. The visual treatment (primary gold button vs. ghost outline) implies two distinct actions. If the contact page has a single form, the user sees the same destination regardless of which CTA they click. Consider differentiating with `?subject=quotation` vs. `?subject=general` query params that pre-fill the form, or collapse to a single primary CTA with the secondary pointing elsewhere (e.g., `/products`, `/about`).
